1. Интерфейс Collection (Списки List и Множества Set)
  Эти методы одинаковы как для ArrayList/LinkedList, так и для HashSet/TreeSet.

- add(E e) — добавляет элемент в коллекцию. Возвращает boolean (для Set вернет false, если элемент уже был).
- remove(Object o) — удаляет элемент. Для списков также есть remove(int index) — удаление по индексу.
- contains(Object o) — проверяет, есть ли элемент в коллекции. Критично помнить: в HashSet это занимает O(1), а в ArrayList — O(N).
- size() — возвращает количество элементов.
- isEmpty() — возвращает true, если коллекция пустая (лучше, чем проверять size() == 0).
- clear() — полностью очищает коллекцию.

2. Специфика Списков (Интерфейс List)
  Методы, завязанные на работу с индексами.

- get(int index) — возвращает элемент по индексу. Помните: в ArrayList это O(1), в LinkedList — O(N).
- set(int index, E element) — заменяет элемент по индексу.

3. Словари (Интерфейс Map)
  У Map своя логика, она не наследуется от Collection, поэтому названия методов отличаются.

- put(K key, V value) — добавляет пару «ключ-значение». Если ключ уже был, перезаписывает значение и возвращает старое.
- get(Object key) — возвращает значение по ключу (или null, если ключа нет).
- getOrDefault(Object key, V defaultValue) — (Любимчик интервьюеров!) Возвращает значение по ключу, а если его нет — дефолтное значение. Заменяет длинные if-else конструкции.
- containsKey(Object key) — проверяет наличие ключа за O(1).
- remove(Object key) — удаляет пару по ключу.
- keySet() — возвращает Set всех ключей (используется для перебора Map).
- values() — возвращает Collection всех значений.

4. Очереди и Стек (Интерфейсы Queue и Deque)
  Для реализации очередей (FIFO) и стеков (LIFO) на собеседованиях всегда используют класс ArrayDeque.

Для Очереди (вставка в конец, забираем с начала):
- offer(E e) — добавляет элемент в конец очереди.
- poll() — извлекает и удаляет элемент из начала очереди. Если очередь пуста, возвращает null.
- peek() — просто смотрит на элемент в начале очереди, не удаляя его.

Для Стека (вставка в начало, забираем с начала):
- push(E e) — кладет элемент на вершину стека.
- pop() — снимает элемент с вершины стека (кидает исключение, если стек пуст).

Подвох, на котором часто ловят: length vs length() vs size()
  Интервьюеры любят смотреть, путается ли кандидат в базовом синтаксисе Java:
- Для обычных массивов (int[] arr): используется свойство arr.length (без скобок).
- Для строк (String s): используется метод s.length() (со скобками).
- Для всех коллекций (List, Set, Map): используется метод .size().

//----------- СТРОКИ ------------------------

1. Строки (String)
  Строки в Java неизменяемы (immutable), поэтому все методы трансформации возвращают новую строку.

- s.length() — получение длины строки (со скобками!).
- s.charAt(i) — получение символа по индексу O(1).
- s.substring(from, to) — взятие подстроки от from (включительно) до to (не включая).
- s.indexOf(str / ch) — поиск первого индекса вхождения символа или подстроки (вернет -1, если не найдено).
- s.equals(otherString) — сравнение строк по значению (никогда не используйте ==).
- s.toCharArray() — превращает строку в массив char[]. Незаменимо, если нужно менять символы местами (в массиве это делать проще).

2. Символы (Character)
  Класс-обертка для char. Все важные методы — статические.

- Character.isLetterOrDigit(ch) — проверка, является ли символ буквой или цифрой (полезно для задач на палиндромы, где надо игнорировать пробелы и знаки препинания).
- Character.isUpperCase(ch) / Character.isLowerCase(ch) — проверка регистра.
- Character.toUpperCase(ch) / Character.toLowerCase(ch) — приведение к регистру.

3. Модификация строк (StringBuilder)
  Используется во всех алгоритмах, где нужно собирать строку по кусочкам в цикле. Обычное сложение через + внутри цикла создает новые объекты и дает сложность O(N²), в то время как StringBuilder работает за O(N).

- sb.append(val) — добавление символа, строки или числа в конец.
- sb.deleteCharAt(index) — удаление символа (критично для алгоритмов со спайдер-бэктрекингом / Backtracking, например, генерация перестановок).
- sb.reverse() — разворачивает строку за O(N).
- sb.toString() — финальная сборка в обычный String.

4. Магия ASCII-кодов (Трюки для Google-интервью)
  Вместо тяжелых HashMap для подсчета частоты английских букв интервьюеры из Google всегда ждут от вас использования обычного массива int[] фиксированной длины (26 или 128) и арифметики символов:

// Перевод символа в индекс массива от 0 до 25 (для 'a'-'z')
int index = itemChar - 'a'; 
// Пример: создание частотного массива букв в строке
int[] counts = new int[26];
for (char c : s.toCharArray()) {
    counts[c - 'a']++; // Идеально по памяти O(1)
}
