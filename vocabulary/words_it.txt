
A couple - «пара», «несколько» или «связанный»
A couple of days — пара дней.
A couple of tasks — пара задач.

"I need a couple of hours to finish the code review." - (Мне нужна пара часов, чтобы закончить проверку кода).


🏃 Управление процессами (Agile / Scrum)

    Backlog — список задач на будущее.
    Sprint — рабочий цикл (обычно 2 недели).
    Daily / Standup — короткий ежедневный созвон.
    Retrospective (Retro) — обсуждение итогов спринта.
    Story points — оценка сложности задачи.
    Velocity — скорость работы команды.
    Roadmap — стратегический план развития продукта.

💻 Разработка и продукт

    Feature — новая функция или возможность.
    Bug — ошибка в коде.
    Fix — исправление ошибки.
    Deploy — перенос кода на рабочий сервер.
    Environment — среда разработки (Prod, Staging, Dev).
    Code review — проверка кода коллегами.
    Refactoring — улучшение структуры существующего кода.
    Legacy — старый, унаследованный код.

🤝 Рабочее взаимодействие (Глаголы)

    To align — согласовать действия или позиции.
    To unblock — помочь устранить помеху в работе.
    To follow up — связаться позже для уточнения статуса.
    To schedule — запланировать встречу.
    To assign — назначить задачу на кого-то.
    To ping — коротко напомнить о себе в чате.
    To escalate — передать проблему вышестоящему руководству.

🛠️ Архитектура, Код и Инструменты

    Boilerplate — шаблонный код, который приходится писать снова и снова.
    Overhead — избыточная нагрузка на систему или память (оверхед).
    Concurrency / Multithreading — конкурентность и многопоточность.
    Garbage Collector (GC) — сборщик мусора; часто обсуждается в контексте падения производительности (GC pauses).
    Memory leak — утечка памяти.
    Dependency Injection (DI) — внедрение зависимостей (база для Spring).
    Pipeline — автоматизированный процесс сборки и деплоя (CI/CD).
    Pull Request (PR) / Merge Request (MR) — запрос на слияние вашего кода с основной веткой.

💬 Технические фразы для созвонов и код-ревью

    "We need to scale this..." — «Нам нужно масштабировать это...» (когда растет нагрузка).
    "This is a breaking change." — «Это изменение ломает обратную совместимость».
    "Let's cover this with unit tests." — «Давай покроем это юнит-тестами».
    "It works locally, but fails in Prod." — «Локально работает, но падает на проде».
    "We should avoid tightly coupled code." — «Нам стоит избегать жестко связанного кода».
    "This approach is deprecated." — «Этот подход устарел (не рекомендуется)».
    "Refactor this to improve readability." — «Перепиши это для улучшения читаемости».
    "Thread-safe" — потокобезопасный (код, готовый к многопоточности).

🐛 Поиск ошибок и Логирование

    StackTrace — снимок цепочки вызовов при ошибке.
    Out of Memory (OOM) — критическая ошибка нехватки памяти.
    NullPointerException (NPE) — классическая «ошибка на миллион долларов» в Java.
    To debug — искать и устранять ошибки в коде шаг за шагом.
    To log — записывать события работы программы в файлы (логи).
