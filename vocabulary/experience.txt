- What is the difference between final, finally, and finalize in Java?
- There are three special words from Java.
final - uses for variables, class or methods and indicates that they cannot be changed after creating.
finally - it is the special word is used at blocks try-catch-finally. Block finally will always execute after blocks try-catch.
finalize - it is the method that used to clean up resources from memory for garbage collector.

- What is the difference between String, StringBuilder, and StringBuffer?
- String - it is the immutable data type in Java. 
If I need to modify a string and work with string I will use type of data StringBuilder.
StringBuilder is used for changing the value of string and concatenating string, for example.
StringBuffer is the same as StringBuilder but thread-safe and is used in concurrent applications. 
StringBuilder is not thread-safe.

- What is the difference between ArrayList and LinkedList in Java?
- ArrayList and LinkedList are implementations of the List interface.
ArrayList is backed by an array. I can get the value from ArrayList for time complexity O(1). 
LinkedList doesn't  have index but it consists of nodes where each node has references to the next and previous elements. 
I can add a new value to LinkedList for time complexity O(1) because it will change references only to left and right values. 
Unlike to ArrayList that will add or delete a new value and shift elements, which takes O(n) time.

- What is the difference between Method Overloading and Method Overriding in Java?
- When I have method in class A, for example, this method called sound() and method sound() in class B. 
And class B extends class A, the method sound() in class B overrides the method from class A. Also this method can have an annotation @Override.
When I have a class with some methods and they all have a name fun1() but all they have different parameter lists - these methods are overloading.
In many cases, it is constructors with diferent count parameters.

Overriding happens between a superclass and a subclass when a method has the same name and signature. Overloading happens within the same class 
when methods have the same name but different parameters, like in overloaded constructors.

- What is the Difference between == and .equals() in Java?
- == is used to compare by references. Method .equals() is used to compare by values.

- What is the contract between equals() and hashCode() in Java?
- There is the contract between equals() and hashCode(), if classes equel bt equals() that classes equel by hashCode().
But if classes equel by hashCode() that doesn't mean classes equel by equals().

The contract states that if two objects are equal according to equals(), they must have the same hashCode(). 
However, if two objects have the same hashCode(), they are not required to be equal according to equals() — this is called a collision.

- What is the difference between Checked and Unchecked exceptions in Java?
- 

Checked exceptions are checked at compile-time. The developer is forced to handle them using try-catch or declare them with throws. 
Unchecked exceptions happen at runtime and inherit from RuntimeException. They usually indicate programming errors and don't require mandatory handling.

- What is the difference between Heap and Stack memory in Java?
- Stack has references to variables and primitive types. 
Heap has other objects and methods.

Actually, it's the other way around. Stack memory stores local variables, primitives, and references to objects, and it's thread-safe. 
Heap memory stores the actual objects themselves, and it is managed by the Garbage Collector.

- Is Java "pass-by-value" or "pass-by-reference"?
- Java is strictly pass-by-value. There is no pass-by-reference in Java.

If you change the object's attributes inside a method (obj.setName("New")), the changes are visible outside because both references 
point to the same object. However, if you reassign the reference inside the method (obj = new Object()), it does not change the original object outside.
